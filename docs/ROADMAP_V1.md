# Official v1.0 开发路线图

本路线图面向产品级落地，不以比赛展示为目标。

---

## 阶段 0：产品仓库初始化

目标：建立 official v1.0 的产品级骨架。

交付物：

- README 产品定位；
- 产品蓝图；
- 模块集成方案；
- 原型迁移清单；
- 目录结构规范；
- 命名规范；
- 初始开发路线。

验收：

- 文档能够解释 official v1.0 与原型仓库的关系；
- 明确哪些能力迁移、哪些能力重构、哪些能力废弃；
- 原型仓库不变。

---

## 阶段 1：后端产品地基

目标：建立可长期维护的模块化后端。

任务：

1. 建立 FastAPI app factory；
2. 建立统一配置系统；
3. 建立结构化日志；
4. 建立数据库迁移版本表；
5. 建立领域模块目录；
6. 建立系统健康检查；
7. 建立统一错误返回；
8. 建立统一审计事件；
9. 建立备份恢复基础接口。

验收：

- `/health` 返回数据库、文件系统、OCR、LLM、向量库、磁盘、版本信息；
- 不再依赖 `from main import app_state` 的跨模块调用；
- 有 `.env.example`；
- 有 Docker Compose；
- 有 Windows 本地启动脚本。

---

## 阶段 2：院务主数据与入住流转

目标：把养老院管理系统底盘搭起来。

任务：

1. Resident 老人中心；
2. FamilyContact 家属联系人；
3. Staff 员工；
4. Bed/Room/Floor/Building 床位房态；
5. CareLevel 护理等级；
6. Admission 入住申请；
7. Assessment 入住评估；
8. Contract 合同；
9. Payment 缴费；
10. Move-in 入住建档；
11. Discharge 离院归档。

验收：

- 可以完成从咨询到入住的完整流程；
- 入住完成后自动生成老人档案；
- 入住完成后自动占用床位；
- 离院后释放床位；
- 每一步都有 timeline；
- PII 字段加密或脱敏。

---

## 阶段 3：护理计划与每日任务

目标：让系统进入养老院日常工作流。

任务：

1. CarePlan 护理计划；
2. CarePlanItem 护理项目；
3. DailyCareTask 每日护理任务；
4. TaskAssignment 任务分配；
5. TaskExecution 任务执行；
6. TaskReview 护士复核；
7. VitalSignRecord 生命体征记录；
8. CareRecord 护理记录归档。

验收：

- 老人可以拥有护理计划；
- 护理计划可以展开为每日任务；
- 护工端可以查看我的任务；
- 任务可以完成、异常、跳过；
- 异常任务进入交接；
- 高风险任务需要护士复核；
- 每日护理记录可按老人/班次查询。

---

## 阶段 4：风险事件与 AI 护理辅助

目标：把原型 AI 任务卡产品化。

任务：

1. Incident 风险事件；
2. IncidentProtocol 护理事件协议；
3. ProtocolVersion 协议版本；
4. ProtocolEngine 协议匹配引擎；
5. AI Task Card Draft 任务卡草稿；
6. Nurse Review 护士审核；
7. RecheckPlan 复测计划；
8. SBAR Draft 交接草稿；
9. CareMemory 护理记忆；
10. RiskSummary 风险摘要。

验收：

- 至少支持 12 类标准风险事件；
- AI 不直接决定高风险任务，协议模板为主；
- 红橙事件必须护士审核；
- 可以生成 SBAR；
- 事件可以关闭和归档；
- AI 输出保存模型版本、协议版本、证据和审核状态。

## 阶段 6：交付与运维产品化

目标：让系统可以交给养老院长期使用。

任务：

1. Windows 安装包；
2. 本地服务控制台；
3. 自动启动与服务守护；
4. 一键备份；
5. 一键恢复；
6. 日志打包；
7. 离线升级包；
8. 升级前自动备份；
9. 升级失败回滚；
10. 授权文件；
11. 轻 SaaS 管控端对接。

验收：

- 非开发者可以启动系统；
- 故障时能生成诊断报告；
- 数据可以备份恢复；
- 升级失败能回滚；
- 断网不影响核心业务；
- 联网只增强授权、升级、远程维护。

## 阶段 7：产品化验收场景

Official v1.0 至少要跑通以下场景：

### 场景 1：新老人入住

咨询登记 → 评估 → 签约 → 缴费 → 分配床位 → 入住建档 → 生成护理计划。

### 场景 2：日常护理

系统生成今日任务 → 护工打卡 → 异常记录 → 护士复核 → 归档护理记录。

### 场景 3：跌倒事件

护工上报 → 协议匹配 → AI 生成草稿 → 护士审核 → 任务执行 → 复测 → SBAR → 归档。

### 场景 4：交接班

系统汇总未完成任务和异常事件 → 生成交接草稿 → 接班确认 → 下一班继续跟踪。

### 场景 5：系统维护

运行诊断 → 备份数据 → 升级系统 → 健康检查 → 失败时回滚。
